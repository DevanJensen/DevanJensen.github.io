The Date Object in JavaScript

We have discussed how object-oriented programming is used to simulate real world objects in our programs. Sometimes we create objects in our programs that represent 'virtual' objects. For example, we have already created an object that represents an email. But an email is not a real thing that you can touch and hold in your hand (unless maybe you consider the printout of an email). Similarly, a date is not something that you can touch, or hold in your hand, but we often use objects in our programs to represent dates. Keeping track of dates is common in programming, that many languages have a 'Date' object built-in. In JavaScript you can create a date object by invoking the Date() constructor function, which is built into the language (you do not have to define the constructor function yourself). You will soon find out that date objects in JavaScript have lots of methods that you can invoke to help you work with dates in your programs.

The Date object is just one of the many tools that you are learning about that can be used to solve problems in software development. You need to know how to use a tool before you can actually use it to fix something. You also need to understand what the tool is capable of, what you can, and can't, do with it. And in many cases, a tool has quirks, and it helps to understand them in order to use it effectively.
