JavaScript Functions

Functions are reusable groups of commands. You can think of them as mini programs within your program.

Let's say we had to convert 10 numbers from celsius to fahrenheit, we could do this...

// convert a bunch of degrees celsius to farhenheit
var f1 = 10 * 9/5 + 32;
var f2 = 50 * 9/5 + 32;
var f3 = 75 * 9/5 + 32;
var f4 = 100 * 9/5 + 32;
//....and so on

But notice how much duplicate code we have written above (repeating code can lead to bugs!).

Let's create a function that converts celsius to farhenheit...

function convertToFarhenheit(tempCelsius){
  var tempFarhenheit = tempCelsius * 9/5 + 32;
  return tempFarhenheit;
}

Note that we start the function definition with 'function', which is a keyword in JavaScript.

Then comes the function name, we can pick any name for the function, but there are some guidelines, for example function names cannot begin with a number.

After the name we put parenthesis. Inside the parenthesis is the parameter, you can use parameters to feed input into your function. Note that the name of the parameter in this example is 'tempCelsius'.

The body of the function goes in between the curly braces. The code that we write in the body of the function will run whenever we use the function.

In the body of the function, we take the parameter and use it in the expression to convert celsius to fahrenheit. Then we return the result. So, parameters allow us to convert any temperature to fahrenheit, which makes this function very flexible. We can pass in any value as a parameter and the function will convert it to degress fahrenheit.

We have defined/declared, a function, but we have not yet used it, let's use it now... 

var f1 = convertToFarhenheit(10);
var f2 = convertToFarhenheit(50);
var f3 = convertToFarhenheit(75);

// lets show the results in the log
console.log(f1);
console.log(f2);
console.log(f3);

Using a function is called invoking, or calling it. In the code sample above, we invoked convertToFahrenheit() three times.

Again, notice how we can pass in any value as the parameter, and in the body of the function we refer to this value by the name of the parameter (in this case, the parameter is named 'tmpCelsius').

Also note that the return value of the function gets assigned as the value of a variable. The third line in the code sample above calls/invokes the convertToFarhenheit() function by passing in 75 and assigning the result (the return value) to the variable f3. When a function returns a value, that value is often assigned to a variable.
For example: var result = someFunction();.

We created the function convertToFarhenheit(), but JavaScript has many functions built into it that we can use, for example alert(), console.log(), and prompt().

Note that you can pass a variable into a function as a parameter: 

var celsiusToday = prompt("What's the temp today (in celsius)?");

var fahrenheitToday = convertToFarhenheit(celsiusToday);

alert("Today's temp in farhenheit is: " + fahrenheitToday);

In the above code sample, we prompt the user to enter today's temperature in celsius, then we store his/her answer in a variable named celsiusToday. Then we feed the variable into the function as a parameter.

Functions can take more than one parameter:

// first we define a function...

function someUselessFunction(param1, param2){
  alert("You passed in the following params:\n" + param1 + "\n" + param2);
}


// then we use it (by calling or invoking it)...

someUselessFunction("sugar", "spice");

Note that functions do not have to have parameters. Also note that functions do not always return something. 

// here we define a function that has no parameters. 
// it also does not return a value...

function doSomething(){
  alert("I'm doing something....(maybe).");
}


// lets call/invoke the function here...
doSomething();

Note that even if a function doesn't have any parameters, you must still include the () when you invoke/call it.

Questions - JavaScript Functions

Question 1

What is a function, and why would you ever want to use one in your code?

A function is a set of commands which are called for when needed. They should be used so you don't need to have the command retyped multiple times.

Question 2

What do you call the 'inputs' that get passed into a function?

They are called parameters.

Question 3

Do all functions have return values?

Not all functions need to return values.

Question 4

What is the 'body' of a function, and what are the characters that enclose the body of a function?

The body of the function is where the commands that will be executed are placed. it is inserted between [ ].

Question 5

What does it mean to 'invoke' a function?

To invoke a function means to call the function and execute it.

Question 6

What does it mean to 'call' a function?

Call is used to change or assign a value when executing the function.

Question 7

If a function has more than one parameter, what character do you use to separate those parameters?

You use an comma to seperate parameters.

Question 8

What is the problem with this code (explain the syntax error)? 

function convertKilometersToMiles(km)
    return km * 0.6217;
}

The function does not have the opening curly bracket({).
