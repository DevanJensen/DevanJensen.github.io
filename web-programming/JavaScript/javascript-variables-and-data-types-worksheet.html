JavaScript Variables and Data Types

Variables

We use variables to store values. Then, instead of repeating the value in your code, you just use the variable. This is handy because if you decide that you want to change the value, you can just update the line of code that 'sets' the value. Then all the lines of code that use the variable will be updated to the new value.

Variables must be declared before they can be initialized. Use the var keyword to declare a variable, and use the assignment operator to initialize a variable (the assignment operator is the equals sign). 

// declare a variable called age, and initialize it to a value of 10...
var age = 10;

The code sample above is a statement. For now you can think of a statement as a single command. Statements in JavaScript end with a semi colon.

Let's assume your want to want show an alert that displays the user's name in various places throughout your program. You could write your program with this approach. 

alert("Bob");

// a bunch of code might go here

alert("Bob");

// maybe a bunch more code might be here

alert("Bob");

Note that in this example we have duplicated Bob's name 3 times. A better approach is to use a variable. That way, if you decide you need to change the variable, to maybe 'Shelly', then you only have to make a change in one place. 

var userName = "Bob";
alert(userName);

// a bunch of code might go here

alert(userName);

// maybe a bunch more code might be here

alert(userName);

Data Types

Variables can hold different data types. Here are variables that store different data types. 

var someNumber = 7;         // as we've already seen, variables can store numbers
var someBoolean = false;    // boolean variables are either true or false
var someString = "Hello!";  // variables can hold 'strings' of characters
var someObject = {};        // we'll talk about objects soon!
var someArray = [];         // we'll talk about arrays soon too!

We'll discuss these data types more in the near future.

Variables that hold boolean values can only be assigned either true or false. Note that true and false are both special keywords in JavaScript.

'String' is a common term that you'll become familiar with as a programmer. A string data type contains characters (letters, numbers, and other characters).

Note that we use quotes around string data types (more on strings soon). Why do you think we have to do this? If you don't use quotes, then JavaScript will think that you are referring to a variable, not a value. 

var day = wednesday;
// JavaScript looks for a variable named wednesday,
// which has not been declared, so this will cause an error


// here is the proper way to do it...
var day = "wednesday";

Can you see the difference between these two variables? 

var x = true;
var y = "true";

Sometimes you will declare a variable but you won't initialize it. 

// you can declare a variable without initializing it to a value
// (it may get a value assigned in another part of the program).
var firstName;

For example, you may let the user assign a value to a variable like this: 

firstName = prompt("Please enter your first name.");
alert("Hello " + firstName);

In this example, we use the prompt() command to ask the user to enter his/her first name. We then store the input in a variable called 'firstName'. We'll learn more about this code sample soon.

Variables can be re-assigned, which is why they are called variables. They can vary. In this example we set a variable to a value, and then we later re-assign a new value to it. 

var day = "Wednesday";
alert(day);
day = "Thursday";
alert(day);

Note that we don't use the var keyword when we re-assign a variable that has already been declared.

Expressions

Expressions are 'equations' that can be solved, such as 5 + 3. This expression evaluates to 8. We can use expressions to assign values to variables. 

var total = 200 + 100;
//the variable 'total' holds a value of 300

var celsius = 40;
var fahrenheit = celsius * 9/5 + 32;

You can actually use expressions with strings, not just numbers. If you add two strings together you get a new string: 

var str1 = "Hello";
var str2 = "World";

var str3 = str1 + str2;

Adding strings together like this is known as string concatenation.

Working with strings is a huge part of programming, and we'll soon see that there are lots of ways to manipulate strings in JavaScript.

Weird things can happen if you combine variables of different data types 

var someNumber = 3;  
var someOtherNumber = "2";
var result = someNumber + someOtherNumber;
alert(result);

You can use the typeof operator to find out the data type of a variable. The typeof operator is a special keyword in JavaScript.

Let's recreate the previous example, but instead of alerting the value of the 'result' variable, let's alert its data type: 

var someNumber = 3;  
var someOtherNumber = "2";
var result = someNumber + someOtherNumber;
alert(typeof result);

Naming Variables

It's important to choose an appropiate name for a variable. It wouldn't make sense to create a variable called 'age' and then store a last name in it.

There are some rules for the characters that can be used in variable names. For example, a variable name cannot begin with a number. There are also other rules, but you'll be asked to explore them on your own in an upcoming assignment. 

// This will cause an error, because variable names are not allowed to start with numbers
var 1stName = "Betty";

Good programmers use camel case when they name variables in JavaScript. You'll also be asked to explain what camel case is in an assignment.

Logging Variables

When you are writing programs, it's often helpful to print out the value of a variable. You can do this by using the console.log() command. For example: 

var lastName = "Jones";
console.log(lastName);

Notice that you put the variable name inside of the parenthesis of the command.
