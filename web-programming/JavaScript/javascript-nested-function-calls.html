Nested Function Calls

In math, when we see lots of parenthesis used in expressions, we evaluate them from the inside out:

x = (7 * (4 * (5 + 3) + 3))

When functions return values, you might see them used inside of other function calls (nested function call). In the previous page (JavaScript Operators) I may have surprised you with some strange code that looked like this: 

var x = parseInt(prompt("Enter a number:"));

If a function returns a value, it can be nested inside another function call when you invoke it. The return value of the nested function call will become the parameter for the outer function call. 

function padWithDashes(str){
    return "---" + str + "---";
}

var someString = prompt("Enter something.");
alert(padWithDashes(someString));

First, we declare a function that takes a string parameter and concatenates dashes before and after the parameter. It also returns the final concatenated string. Then we prompt the user to enter a string and store the response in someString.

The final line is what we are interested in, here we see 'nested' function calls. The return value of the call to padWithDashes() is passed as the parameter to alert().

There is a script on line 11 of this file, go ahead and try it out.

We could actually condense this code even more. See if you can wrap your mind around this code: 

alert(padWithDashes(prompt("Enter something.")));

Just remember that nested function calls are evaluated from the inside out. That is, the inner most function call is executed first and it's return value is passed as the parameter to the next function call.

Also remember that you when you nest function calls like this, they should be functions that return a value.
