The Object Data Type in JavaScript

We previously talked about variables, and how they store different types of data. We discussed numbers, strings, and boolean data types. Another data type that a variable can store in JavaScript is called the 'object' data type.

When writing programs, we often need to simulate real world objects. For example we may need to create a program that keeps track of dogs (we got hired to build a website for the Humane Society). So we might create 'dog' objects in our code. A dog object should contain all the information about a dog that we need for the purposes of our program. So if our program needs to keep track of a dog's name and its breed, then each dog object that we create in our code should store that information. So 'name' and 'breed' would be the properties of a dog that we are interested in keeping track of.

An object is a collection of properties and their values.

Consider the properties of a book? Size, publisher, number of pages, author, title, and many more. What are the values that can be assigned to these properties? Note that the values of these properties may be different data types. If an object has an 'age' property, the value would be a number, a 'name' property would probably have a string value.

Object-oriented programming is all about simulating real world objects in a program!

We can use objects in our programs to model, or simulate, things in the real world. These 'virtual objects' can represent many things, for example if we are writing software for a library, we would probably have many 'book' objects that represent the data we need to keep track of each real book in the library. Our program might even keep track of many libraries, so each library object may contain it's own collection of books. The possibilties for the types of objects that you can create is endless!

Just like objects in the real world, objects in our programs can be extremely complicated, we'll see some complex objects when we discuss the DOM in a little while.

For now, all you need to know is that objects have properties (we'll learn a little more about objects later in the course). Here is how we can create an object in JavaScript code: 

var myDog = { name: "Lilo",  breed: "mut",  age: 3,  vaccinated: true };

This object has 4 properties (name, breed, and age). Each one of these properties is set to a value (name = Lilo, breed = mut, age = 3).

Notice the syntax; there's a lot going on in this simple little object!

  • We use curly braces to enclose the properties of the object
  • Each property is separated from its value by a colon. Each property and its value is called a property/value pair (some programmers say key/value pairs rather than property/value pairs
  • Property/value pairs are separated by commas (but there is no comma after the last property/value pair)
  • Note that the 'age' property value (3) does not have quotes around it because its data type is a number, not a string.
  • Also note that the 'vaccinated' property value does not have quotes around it because it is a boolean data type (and that true is a special keyword in JavaScript)

Sometimes you see objects written on multiple lines, which is common for objects that have lots of properties: 

var myDog = {
  name: "Lilo", 
  breed: "mut", 
  age: 3,
  vaccinated: true
};

The object above is assigned to a variable named myDog. We can access the value of a property by using the dot operator. 

// Let's log each property of myDog...
console.log( myDog.name );
console.log( myDog.breed );
console.log( myDog.age );

We can also change the values of properties by using the dot operator, like so:

myDog.name = "Buster";
myDog.breed = "Boxer";
myDog.age = 7;
myDog.vaccinated = false;

In the code snippet above, we have changed each of the properties of the myDog object (we have changed the values of each property).

JavaScript allows you to create an empty object, and then add properties to it later: 

// create an empty object
var person = {};


// then add a property to it
person.firstName = "Ted";

There is an alternative syntax for getting and setting the values of properties, by using square brackets like this (this is called bracket notation while using the dot operator is called dot notation). 

myDog['age'] = 10;
console.log( myDog['age'] );

When working with objects, we usually use dot notation, but we'll be using bracket a lot when we start working with arrays soon.

Note that when using bracket notation, the property name inside the square brackets must be enclosed in quotes (either single quotes or double quotes). If you do not use quotes inside the brackets, then whatever is inside the brackets will be evaluated as an expression, which can come in handy some times. For example: 

var myDog = {
  name: "Lilo", 
  breed: "mut", 
  age: 3,
  vaccinated: true
};

var x = prompt("Which property of myDog would you like to look up?");
console.log( myDog[x] );

// I encourage you to run this code sample a few times
// When the prompt appears enter a valid property name of the myDog object
// Then run it again and enter something that is not a valid property name, see what happens.

Here's a common beginner mistake when using bracket notation (forgetting to use quotes): 

console.log( myDog[breed] );

// without the quotes inside the brackets, JS looks for a variable named 'breed'.
// if you didn't declare a breed variable before this line of code, then it doesn't exist and you'll get an error.

Objects are a critical part of many programming languages. They allow us to make models that simulate real-world objects in our programs. We'll soon find out that objects can do much more as well.
