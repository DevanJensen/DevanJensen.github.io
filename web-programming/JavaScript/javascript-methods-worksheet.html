JavaScript Methods

This program that you are enrolled in focuses heavily on creating data-driven programs. These are programs that keep track of data and allow users to view data, add data, edit data, and in some cases delete data. In a previous lesson we discussed how we can use objects to keep track of data. We used the example of a 'dog' object that keeps track of the name and breed of each dog in our program. So we can simulate dogs by creating objects that have 'name' and 'breed' properties.

Objects in our code can also be used to simulate the behavior of real-world objects. What are the behaviors of a real dog? They bark, walk, eat, sleep, etc. In order to simulate these behaviors in our programs, we can add functions to our dog objects.

Let's consider what a dog object could look like if we start adding behaviors: 

var dog = {
  name:  "Lassie",
  breed: "Border Collie",
  bark: function(){
    alert("Ruff!");
  },
  eat: function(amount){
    console.log("Just ate this much: " + amount);
  }
};

It takes a little bit of getting used to to sort through this syntax. What we have done is added two new properties, bark, and eat. The values of the new properties are functions that simulate the behavior of a dog.

If an object has properties that are functions, we can invoke the functions by using the dot operator, just as we do when access other properties of an object. The difference is that we must add the parenthesis () in order to invoke the method. Recal that when we invoke functions, we put () after the function name. 

// to call/invoke a method of an object, you can do it like this:
dog.bark();
dog.eat(10);

When properties of objects happen to be functions, then we call them methods.

Methods can access other properties inside the object, in order to do so, you have to use 'this', followed by the dot operator, followed by the name of the property that you wish to acess. Consider this object that simulates an purchase order: 

var purchaseOrder = {
  customerName: "Bob Smith",
  productName: "Bicycle",
  productPrice: 199.95,
  print: function(){
    console.log(this.customerName);
    console.log(this.productName);
    console.log(this.productPrice);
  }
};

// let's invoke the print() method of the purchase order
purchaseOrder.print();

In a previous worksheet we used a 'student' object as a prameter for functions what we wrote (calculateAverageGrade and calculateFinalGrade). Since these functions were specifically created to work with student objects, we could turn them into methods of the student object. Now our student object is a logical grouping of all the properties and methods for working with students. 

var studentObj = {
    
    // properties
    name:         "Bob",
    midTermExam:  88,
    finalExam:    92,

    // methods
    calculateAverageGrade: function(){
      return (this.midTermExam + this.finalExam)/2;
    },
    
    calculateFinalGrade: function(){
      
      var avg = this.calculateAverageGrade();
    
      var finalGrade;

      if(avg >= 90){
          finalGrade = "A";
      }else if(avg >= 80){
          finalGrade = "B";
      }else if(avg >= 70){
          finalGrade = "C";
      }else if(avg >= 60){
          finalGrade = "D";
      }else{
          finalGrade = "F";
      }

      return finalGrade;
    
    }
};

Questions - JavaScript Methods

Question 1

How is a method different from a regular function in JavaScript?

The function only works within the object, so it's referred to as a method.

Question 2

Why would we want to add methods to an object?

It can save us from having to rewrite the same lines of code multiple times.

Question 3

How can we access the property of an object from inside the body of a method of that object?

You invoke it by using using something such as "letter.print()", letter being the name of the object and print being the name of the method.
